# Copilot Token Usage

Track GitHub Copilot CLI token consumption and premium request spend from local session
data. **Copilot CLI only** — reads `session.shutdown` events written automatically to
`$COPILOT_HOME/session-state/*/events.jsonl`. No configuration required.

## When to use this skill

Use when the user asks about:
- Copilot token consumption (input, output, cache-write, cache-read, reasoning)
- Premium request usage or counts
- Which model was used most
- Daily/weekly/monthly usage breakdowns
- Billing impact or cost estimation

## How to use this skill

### 1. Determine the reporting period

| User asks about | `--period` flag |
|-----------------|-----------------|
| today | `--period=day` |
| this week / last 7 days | `--period=week` (default) |
| this month / last 30 days | `--period=month` |
| last N days | `--days=N` |

### 2. Locate the report script

```bash
# Discover COPILOT_HOME (try env var, then standard paths)
if [ -n "${COPILOT_HOME:-}" ] && [ -d "$COPILOT_HOME/session-state" ]; then
  :
elif [ -d "$HOME/.copilot/session-state" ]; then
  COPILOT_HOME="$HOME/.copilot"
else
  COPILOT_HOME="${XDG_CONFIG_HOME:-$HOME/.config}/copilot"
fi
SCRIPT="$COPILOT_HOME/installed-plugins/e-roux-plugins/billing/scripts/report.sh"
```

### 3. Run the report

```bash
bash "$SCRIPT" --period=<day|week|month>
# or
bash "$SCRIPT" --days=<N>
```

### 4. Present the output

Highlight to the user:
- **Premium requests** — the GitHub billing metric (`PREM` column)
- **Cache-write tokens** — these are billed at higher rates (typically 3.75× input price)
- **Model breakdown** — sorted by premium cost, most expensive first
- If `CACHE-W ≈ INPUT`: normal — first call in a context writes the full context to cache

## Data source and reliability

- Source: `session.shutdown` events in `$COPILOT_HOME/session-state/*/events.jsonl`
- Written automatically at clean session exit — no env vars or config needed
- Crashed sessions (no `session.shutdown`) fall back to `assistant.message` output tokens only
- The `sessionEnd` hook (included in this plugin) additionally caches data to
  `$COPILOT_HOME/plugin-data/e-roux-plugins/billing/usage.jsonl` for faster access

## Token types explained

| Field | Meaning |
|-------|---------|
| `INPUT` | Tokens sent to the model (prompt + context) |
| `OUTPUT` | Tokens generated by the model |
| `CACHE-W` | Context written to cache on first call — billed at higher rates |
| `CACHE-R` | Context read from cache on subsequent calls — cheap |
| `REASON` | Extended thinking / reasoning tokens |
| `PREM` | Premium request units — the actual GitHub billing metric |
